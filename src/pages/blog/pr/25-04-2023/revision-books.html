Althought it does not look so complicated, there are a lot of little quirks that took me some time. First of all, this software is, as many others, used by many, people, so I had to implement login and login management system. Initially, I wanted to use some plug-and-play solution like Microsoft's Azure Active Directory, however this software has to run even without internet, so that's not an option. So, instead, I implemented one myself.

Remember the part where it has to run locally? Yeah, I had to implement everything locally. That means that the API and Database are running locally as well :D. Files are stored locally as well. That's not good, because when something fails (hardrive dies, or whatever, also, let's ignore the fact that they're in a RAID), you don't have backups, however Azure provides a service called Azure Backup, which solves that issue.
